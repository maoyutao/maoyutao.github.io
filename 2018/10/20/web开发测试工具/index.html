web开发测试工具

前言

在web开发过程中，好的工具可以使我们提高效率，事半功倍。这次抢票平台开发过程中，我主要使用了以下三种工具来辅助开发，现在写一点对它们的总结。

postman

postman是用来测试API的，在后端开发的时候可以减少对前端的依赖性，自己测试。

这是它的界面，可以说它的设计还是十分友好的，就算没用过这个工具的人也一看界面就大概知道如何使用了。

左边是请求的历史和保存的请求集。请求集不仅可以把请求比较清楚地整理在一起，还可以生成postman平台的连接来分享给别人，很适合团队合作。

中间部分是主要部分，指定请求方式、请求路径、请求参数之后点击send就可以发送请求，并在下面查看结果。postman对cookie等都有很友好的支持～

点击右边的code可以快速生成不同语言的代码，下面这张图生成的就是python的代码。

右上角还有环境的设置，这个功能还没有使用过，以后再慢慢探索吧～

jmeter

jmeter也是发送请求的工具，但特点是能同时发送很多请求，因此常被用来做压力测试。

这是它的界面：

下面用微信抢票测试的例子来说明它的部分用法。

新建线程组

首先在test plan下新建一个线程组

这是线程组的一些设置，当前的线程数被设置成了3000

添加HTTP请求

然后在线程组中添加一个http请求

给http设置域名、端口、路径、请求方法和参数等。可以看到这里的参数除了普通的值还有用\${}包起来的东西，这个是jmeter里的变量，可以用户自定义或者是用函数生成，还可以从csv中导入。这里的open_id和stu_id就是从csv中导入的。

从csv中导入变量

在线程组下添加一个 CSV Data Set Config

variable names一栏填变量名，csv中有几列就填几个，它会每次读一行给分给一个线程，Recycle on EOF就是指在线程数大于行数时，读完之后是否运行从头开始读。

这样配置之后就可以在其他地方使用变量，用\${}包起来。

发送请求

配置完了之后点击页面上方的绿色按钮就可以发送请求了

查看结果

jmeter和postman不一样，不是直接看到结果的，需要在HTTP下面添加结果查看器。我喜欢添加View Result Tree和View Result in Table两种。

View Result Tree可以看到某个具体请求的返回结果

而View Result in Table可以看到所有请求的一个总体情况

还有summary Report可以看到测试结果的总体数据

中文问题

使用过程中被编码问题坑了很久，问题分成两种情况，一是发送的参数中有中文，服务器接受到的是乱码，一种是服务器返回的内容有中文，这边显示的返回结果是乱码。

  • 问题1的解决方案

    在HTTP的设置中将content-encoding设置成utf-8

  • 问题2的解决方案

    在HTTP请求下增加BeanShell PostProcessor，并在script处填入
    prev.setDataEncoding("UTF-8");

每个线程循环请求

jmeter还能添加逻辑控制器，如loop controller，switch controller等

travis CI

travisCI是提供持续集成服务的工具，它绑定 Github 上面的项目，只要有新的代码，就会自动抓取。然后，提供一个运行环境，执行测试，完成构建，还能部署到服务器。

免费的版本只支持公开库，收费的版本才支持私有库，虽然说好像有50次免费使用的机会，但我一直打不开私有版本的库，只好作罢。（org免费，com收费）

travis CI网站上只会列出你所有的库，而不能是作为协作者的库，但亲测先转让库然后配置了travis之后再转回去之后还是可以在travis的网站上看到项目。

Travis 要求项目的根目录下面，必须有一个.travis.yml文件。这是配置文件，指定了 Travis 的行为。

我们的微信抢票项目用的是django，需要用到travis平台上的mysql，使用这个有一个坑就是要修改配置文件使Django 能正常地连接到travis上的mysql。travis CI的mysql用户和密码是root和空，需先将configs.json的数据库用户配置改成root和空，然后进行加密操作得到configs.json.env，再将configs.json.env上传。但configs.json仍然保持被ignore，本地仍然加载configs.json，但travis CI会使用configs.json.env解密得到的configs.json，实现travis CI测试配置和本地配置分离。

这是我们的.travis.yml配置文件

它做了这么几个事：

  • 指定python和django的版本
  • 指明需要mysql服务
  • 运行前安装必要的库
  • 解密得到configs.json文件
  • 运行测试
  • 指定测试结果的通知方式
