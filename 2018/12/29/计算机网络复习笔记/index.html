计算机网络复习笔记

Chapter 1: introduction

What is the Internet?

network edge: hosts, access net, physical media

network core: packet/circuit switching, Internet structure

transmission rate = bandwidth（bits per second）

ISP(Internet Service Provider)

Internet structure: network of networks

Internet standards:

  • RFC: Request for comments
  • IETF: Internet Engineering Task Force

protocols define format, order of msgs sent and
received among network entities, and actions taken on msg transmission, receipt

network edge

How to connect end systems to edge router？

  • residential access nets（住宅）
  • institutional access networks (school, company)
  • mobile access networks

dedicated access

  • Dial-up Modem（拨号上网）
    • Uses existing telephony infrastructure
    • Can’t surf and phone at same time: not “always on”
  • Digital Subscriber Line (DSL)（数字用户线路）
    • Also uses existing telephone infrastruture

share access

  • Residential access: cable modems （有线电视网络 电缆）
  • Fiber to the Home（光纤）
    • Much higher Internet rates
  • Ethernet Internet access（以太网）
    • Typically used in companies, universities, etc
    • 每个节点必须获取电缆或者信道的才能传送信息，有时也叫作以太（Ether）
  • Wireless access networks
    • via base station aka（as known as) “access point”
    • wireless LANs:802.11b/g (WiFi)
    • wider-area wireless access（广域网 3G）

典型的家庭网络

Physical Media

  • Coaxial cable（同轴电缆）
  • Fiber optic cable（光纤）
  • radio

network core

how is data transferred through net?

  • circuit（电路）-switching: dedicated circuit per call: telephone net
    • 预留资源
  • packet-switching: data sent thru net in discrete “chunks”
    • store and forward

Internet structure: network of networks

  • at center: “tier-1” ISPs: treat each other as equals(13台)
  • “Tier-2” ISPs: smaller (often regional) ISPs
    • Connect to one or more tier-1 ISPs, possibly other tier-2 ISPs
  • “Tier-3” ISPs and local ISPs
    • last hop (“access”) network (closest to end systems)

delay、loss and throughput

Delay in packet-switched networks

  • proc processing(计算、检验、决定去处的算法执行时间）
  • queueing（在路由器排队等候）
  • Transmission delay（路由器发出的时间 L/R）
  • Propagation delay（传播的时间 d/s）

Throughput：rate (bits/time unit) at which bits transferred between sender/receiver

Internet protocol stack



Chapter 2: Application Layer

principle

Network core devices do not run user application code

Application Architectures

  • Client-server
  • Peer-to-peer
  • Hybrid of client-server and P2P

Sockets：Process sends/receives messages to/from its socket

app requirements

  • loss
  • time
  • bandwidth

App-Layer Protocol Defines

  • Types of messages exchanged
  • Message syntax
  • Message semantics
  • Rules for when and how processes send & respond to messages

web and HTTP

User agent for Web is called a browser

HTTP: hypertext transfer protocol

Uses TCP

HTTP is “stateless”

  • Nonpersistent HTTP
    • At most one object is sent over a TCP connection
    • HTTP/1.0 uses nonpersistent HTTP
  • Persistent HTTP
    • Multiple objects can be sent over single TCP connection between client and server
    • HTTP/1.1 uses persistent connections in default mode

RTT: time to send a small packet to travel from client to server and back

Two types of HTTP messages: request, response


User-Server State: Cookies

Web Caches (Proxy Server)：

  • Reduce response time for client request.
  • Reduce traffic on an institution’s access link.
